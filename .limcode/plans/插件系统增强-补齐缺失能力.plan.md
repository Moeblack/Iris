# 插件系统增强计划

## 目标
补齐插件系统的 6 大缺失能力，使插件从"功能扩展器"升级为"完整的切面拦截器"。

## 改动文件清单

### 1. `src/plugins/types.ts` — 类型定义（核心）
- PluginHook 新增 5 个钩子：`onBeforeLLMCall` / `onAfterLLMCall` / `onAfterToolExec` / `onSessionCreate` / `onSessionClear`
- 新增 3 个拦截器类型：`BeforeLLMCallInterceptor` / `AfterLLMCallInterceptor` / `AfterToolExecInterceptor`
- IrisAPI 扩展：加入 `config` / `mcpManager` / `computerEnv` / `ocrService`
- PluginContext 新增 `getRouter()` 方法

### 2. `src/llm/router.ts` — LLM 路由器
- 新增 `registerModel()` 和 `unregisterModel()` 公开方法，支持动态注册/移除模型

### 3. `src/plugins/context.ts` — 插件上下文实现
- 实现 `getRouter()` 方法

##/core/tool-loop.ts` — 工具循环
- ToolLoopConfig 新增 `beforeLLMCall` / `afterLLMCall` / `afterToolExec` 拦截器字段
- run() 中在 prompt.assemble() 之后、callLLM 之前调用 beforeLLMCall
- run() 中在 callLLM 之后、push history 之前调用 afterLLMCall
- executeTools() 中透传 afterToolExec

### 5. `src/tools/scheduler.ts` — 工具调度器
- executeSingle() 新增 afterToolExec 参数
- 在工具执行完成后调用 afterToolExec
- executePlan() 透传 afterToolExec

### 6. `src/core/backend.ts` — 后端
- setPluginHooks() 中组合新钩子为拦截器并注入 toolLoopConfig
- handleMessage() 中调用 onSessionCreate（首次消息时）
- clearSession() 中调用 onSessionClear

### 7. `src/bootstrap.ts` — 启动流程
- 将 r入 PluginManager.loadAll()
- notifyReady() 传入完整组件（config / mcpManager / computerEnv / ocrService）

### 8. `src/plugins/manager.ts` — 插件管理器
- loadAll() 签名扩展，接收 router
- PluginContextImpl 构造传入 router
